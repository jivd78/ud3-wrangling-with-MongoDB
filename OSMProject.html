1.2 Non Consistent Postal Codes

                                                

Another way to locate an establishment is with its postal code. This attribute is preferred over an address itself, since, in theory, its uniformity is better than other address atributes uniformity, like streets, etc. Unfortunately, Sao Paulo post codes are formed by 8 digits, separeted or not by a hifen at fifth digit. Since this digit is correct, but not always produced, we decided to clean all postcodes, only remaining numeric characters.

this function cleans up post code issues:

                                                 

			
def post_code_treatment(postcode_value):
	"""
	This function substitues non alphanumeric characters ("."," " and "-" ) by no 
	character at all. Postcodes are transformed into numeric strings only.
	"""
	pv = postcode_value
	translated_str = pv.replace("-","")
	translated_str = translated_str.replace(".","")
	translated_str = translated_str.replace(" ","")    
	return translated_str}

This query let us analyse the final result of postcode cleaned up:

3. Additional Ideas

                                                

3.1. How to improve input data Volume.

There is no point of analysing data if this is not enough to get minimum insights.

                                                

For instance, in a city with at least a pair of million of stablishments, must exists a way to improve significant data input of node type data.

                                                

It is very relevant that 50% of all documents were input by only 10 users. Furthermore, 311 out of 1603 users input nothing more than a single document.

                                                

It is a fact that OSM data and contributors are at levels far below a Data Scientist would expect in order to produce productive insights. It is also known that all licensing from OSM are for free, and all contributions are voluntary. I am afraid this is a major cause of low contributions. On the other hand, it is not a mistery that economic purposes can spark dynamics in commercial relations. Then, what if we can manage a way to atract and let for profit agents to use and modifiy GPS data from OSM platform?. For agents I mean non physical person but corporations, and institutions. I believe this kind of contributors can increase significantly the current database. Those agents must be mainly contributors that need an open platform like OSM. They are going to use their own inputs and get them free for the entire community.

                                                

The biggest OSM draw is the for free complete availability of raw data, perfect for developers, in contrast with Google Maps licensing, and that must be the strogest concept when pitching the idea to possible contributors-corporations.

                                                

Although this is already happening, Google dominant position is a difficult barrier to surpass. I believe marketing and its related cost are the most significant task to tackle in order to increase data volume contributions.

5. Conclusion

                                                

There are lot of value in geographical and localization data, even more if this is gathered toghether with extra data about the points tagged. Databases produced from this data could generate powerful insights and have economic potencial. All this is possible only if this data is highly populated. In the case this db lack enough information, for sure, any insight could lead to wrong conclusions or strategies, since data is not a representative sample of the entire population. Marketing Open Street Data capabilities for new geographical support apps would be part of the solution, offering as main feature its free platform.
